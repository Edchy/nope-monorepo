# NOPE Blog

## Hard rules

- Keep changes simple and composable, and solve root causes instead of symptoms.
- Plan before work with 3+ steps or architectural impact.
- Verify before done: tests pass when relevant, the build succeeds, and the UI looks correct.
- Use design tokens instead of hardcoded style values.
- Log durable mistakes or project-specific traps in `gotchas.md`.
